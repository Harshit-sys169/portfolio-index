# GitHub Discussions Quick Reference

## What are GitHub Discussions?

Discussions are a way for communities to ask and answer questions, share announcements, and have conversations. They're like a hybrid between issues (structured) and forums (casual).

## How to Enable Discussions

### For Each Repository:

1. **Navigate to Settings**
   - Go to repository → Settings tab (gear icon)
   - Must have admin access

2. **Find Features Section**
   - Scroll down to "Features" section
   - Look for "Discussions" checkbox

3. **Enable Discussions**
   - Check the "Discussions" box
   - GitHub creates default categories automatically
   - Click "Save" if prompted

4. **Access Discussions**
   - New "Discussions" tab appears in repo
   - URL: `https://github.com/[owner]/[repo]/discussions`

## Discussion Best Practices

### Create Welcome Post
```markdown
# 👋 Welcome!

Hello! This is a space to discuss [Project Name].

**Feel free to:**
- 💬 Ask questions about the methodology
- 🤔 Discuss potential improvements
- 📊 Share your own applications
- 🔧 Suggest extensions or modifications

**Guidelines:**
- Be respectful and constructive
- Search for existing answers first
- Include relevant code/data when possible
- No spam or self-promotion
```

### Set Discussion Categories

**Recommended categories:**
- **Announcements** - Major updates
- **General** - Off-topic discussions
- **Ideas** - Feature/improvement suggestions
- **Polls** - Community feedback
- **Q&A** - Questions and answers
- **Show and Tell** - Showcase your work using this project
